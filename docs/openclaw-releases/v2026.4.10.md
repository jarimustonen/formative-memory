# v2026.4.10

**Julkaistu:** 2026-04-11
**Vaikutus:** 🟡 Kohtalainen — Active Memory on komplementaarinen pipeline-plugin (ei kilpailija), Codex-provider

## Pluginiin vaikuttavat muutokset

- **Memory/Active Memory -plugin** (#63286) — Uusi valinnainen Active Memory -plugin: proaktiivinen pre-reply muisti-subagent joka ajaa juuri ennen päävastausta ja injektoi relevanttia kontekstia piilotettuna systeemikontekstina.

  **Arkkitehtuuri:** Blocking sub-agent jolla on käytössä vain `memory_search` ja `memory_get`. Se hakee muistoja kulloinkin aktiiviselta memory-slot-pluginilta ja palauttaa tiivistelmän tai `NONE`. Tiivistelmä injektoidaan `<active_memory_plugin>`-tageilla untrusted prompt-prefix -polulle.

  **Suhde meidän pluginiin:** Active Memory **ei kilpaile** `plugins.slots.memory`-slotista. Se on erillinen pipeline-plugin (`plugins.entries.active-memory`) joka toimii minkä tahansa memory-slotin rinnalla. Jos käyttäjä asentaa sekä meidän associative-memoryn (slotti) että Active Memoryn (pipeline), subagent hakee muistoja meidän tietokannastamme. Nämä ovat komplementaarisia:

  | | `plugins.slots.memory` (meidän plugin) | Active Memory (pipeline) |
  |---|---|---|
  | Rooli | Muistin tallennus, haku, konsolidaatio, dreaming | Proaktiivinen pre-reply recall |
  | Eksklusiivisuus | Vain yksi kerrallaan | Toimii minkä tahansa slotin rinnalla |
  | Tallennus | SQLite, assosiaatiot, temporaalinen tila | Ei tallenna mitään pysyvästi |

  **Konfiguraatio:** `queryMode` (message/recent/full), `promptStyle` (balanced/strict/contextual/recall-heavy/precision-heavy/preference-only), `timeoutMs`, `maxSummaryChars`, `cacheTtlMs`, `thinking`-taso, prompt-overridet. `/verbose on` ja `/trace on` debuggaukseen.

  **Mahdollisuus:** Meidän plugin voisi hyötyä Active Memoryn proaktiivisesta recall-kerroksesta. Voisimme dokumentoida suositellun konfiguraation jossa molemmat ovat käytössä yhdessä.

  **Lisätiedot:** [docs.openclaw.ai/concepts/active-memory](https://docs.openclaw.ai/concepts/active-memory), [openclaw/openclaw#63286](https://github.com/openclaw/openclaw/pull/63286)

- **Models/Codex provider** (#64298) — Bundlattu Codex-provider: `codex/gpt-*`-mallit käyttävät Codex-hallittua authia, natiiveja threadeja, model discoveryä ja compactionia. **Vaikutus:** Ei suora vaikutus pluginiin. Uusi provider-vaihtoehto jota käyttäjät voivat käyttää — embedding-resoluutiomme toimii provider-agnostisesti.

- **Gateway: `commands.list` RPC** (#62656) — Remote gateway -clientit voivat nyt discoveroida runtime-native, text, skill ja plugin -komentoja. **Vaikutus:** Meidän `/memory-init`, `/memory-status` yms. komennot ovat nyt löydettävissä remote-clienteille automaattisesti. Ei vaadi muutoksia.

- **Models/providers: `allowPrivateNetwork`** (#63671) — Per-provider `models.providers.*.request.allowPrivateNetwork` self-hosted OpenAI-yhteensopivien endpointtien luottamiseen. **Vaikutus:** Mahdollisuus käyttäjille jotka ajavat lokaaleja embedding-malleja (esim. Ollama privat-verkossa). Ei vaadi muutoksia pluginilta.

- **Browser/security-parannukset** — Laaja SSRF-suojausten tiukennus (hostname allowlistit, interaction-driven redirectit, CDP discovery, Docker CDP source-range). **Vaikutus:** Ei suora vaikutus — pluginimme ei käytä browser-toiminnallisuutta.

- **Security/tools-hardening** — Exec preflight reads, host env denylisting, node output boundaries, outbound host-media reads, profile-mutation authorization, plugin install dependency scanning. **Vaikutus:** Dependency scanning saattaa vaikuttaa pluginin asennusprosessiin. Meidän plugin on puhdas (ei haitallisia riippuvuuksia), joten ei ongelmaa.
